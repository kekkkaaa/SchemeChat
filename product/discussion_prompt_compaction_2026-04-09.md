# Discussion Prompt Compaction 2026-04-09

## 背景

首轮 Draft 之前采用“轮次说明 + 本轮目标 + 输出要求 + 常驻规则逐条展开 + 临时补充”的长模板。

这类模板会带来两个问题：

- 模型要在生成每个 token 时持续参考一大段重复约束，明显拖慢回复速度
- 大量重复规则会稀释题目本身，增加 prompt 噪音

## 本次调整

首轮 Draft 改为短模板，只保留 4 类必要信息：

1. 当前轮次身份
2. 题目本身
3. 固定输出结构
4. 压缩后的规则摘要与临时补充

新模板形态：

```text
第 1 轮独立分析，不回应其他 AI。
题目：...
请用 1/2/3 输出：1. 核心结论 2. 关键依据或判断路径 3. 主要不足、风险或最不确定的一点。
要求：高压缩；先结论后理由；独立判断，不迎合；用 1/2/3；便于其他 AI 转述。
补充：仅保留本轮新增限制。
直接开始，不寒暄，不复述题目。
```

## 装配原则

- 常驻规则不再逐条展开成长句，改为短语摘要
- 快捷提示不再重复“这轮请……”句式，改为短语摘要
- 用户临时补充仍保留原文，避免损失用户明确输入
- 默认规则仍然保留，但不再用长段落重复表达

## 预期收益

- 缩短首轮 Draft 长度
- 降低模型处理负担
- 提高首字出现速度和流式稳定性
- 保持轮次控制语义不变，只压缩表达层
