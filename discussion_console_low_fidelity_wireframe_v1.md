# SchemeChat 讨论控制台低保真原型说明 V1

## 文档目的
本文档用于把前面的产品逻辑进一步收束成“可直接画原型”的低保真说明。

本文档重点描述：

- 桌面端主界面的整体排布
- 底部讨论控制台在不同阶段的骨架
- 不同状态下哪些区域高亮、哪些区域只读
- 主按钮、辅助按钮、状态反馈的相对位置

本文档不讨论视觉风格细节，不讨论代码实现。

---

## 一、原型原则

### 1. 先保证结构清晰，再谈视觉表现
V1 原型的重点不是做得花，而是让人一眼知道：

- 现在在第几轮
- 这一轮要干什么
- 本轮 Draft 在哪里看
- 下一步点哪里

### 2. 保持“原网页轨道 + 控制台”双层结构
主界面不重做成复杂 SaaS 台面，而是保留：

- 上方或中部：多个 AI 官方网页轨道
- 下方：一块固定的讨论控制台

### 3. 控制台永远固定在底部
不管在哪个状态，底部控制台都应固定存在。

变化的是：

- 控制台里哪个区域更强调
- 哪些内容展开
- 主按钮文案如何变化

### 4. 始终只保留一个最强主按钮
用户在任何状态都应该一眼看到唯一主动作。

---

## 二、桌面端整体骨架

## 1. 页面总体布局

建议桌面端采用纵向三层结构：

1. 顶部会话总览条
2. 中部 AI 轨道区
3. 底部讨论控制台

### 低保真结构

```text
+----------------------------------------------------------------------------------+
| 顶部会话总览条                                                                   |
| 讨论标题 | 第 2/4 轮 | 4轮标准 | 3个参与AI | 临时对话已开启 | 设置               |
+----------------------------------------------------------------------------------+
| 中部 AI 轨道区                                                                    |
| +----------------+ +----------------+ +----------------+                         |
| | ChatGPT        | | Gemini         | | Grok           |                         |
| | 发言 / 已完成  | | 发言 / 生成中  | | 静默 / 本轮跳过 |                         |
| |                | |                | |                |                         |
| | 官方网页内容    | | 官方网页内容    | | 官方网页内容    |                         |
| |                | |                | |                |                         |
| +----------------+ +----------------+ +----------------+                         |
+----------------------------------------------------------------------------------+
| 底部讨论控制台                                                                    |
| [阶段/轮次/目标]                                                                  |
| [规则Tag][临时补充Tag][参与者状态]                                                |
| [Draft标题][来源入口][版本状态]                                                   |
| [Draft正文编辑区..............................................................]   |
| [快捷文本][常驻规则][重生成草稿]                             [发送交叉讨论]       |
| [状态反馈 / 异常提示 / 系统建议]                                                  |
+----------------------------------------------------------------------------------+
```

---

## 2. 顶部会话总览条

### 作用

- 提供全局会话信息
- 不承载复杂操作

### 推荐内容

- 讨论标题
- 当前轮次
- 当前模式
- 当前参与 AI 数量
- 临时对话状态
- 一个轻量设置入口

### 原型要求

- 高度较低
- 信息一行排开
- 不应喧宾夺主

---

## 3. 中部 AI 轨道区

### 作用

- 保留真实网页交互
- 展示每个 AI 当前轨道状态

### 每个轨道顶部都应有轻状态栏

建议状态栏包含：

- Provider 名称
- 当前角色
- 当前轮状态

例如：

- `ChatGPT | 发言 | 已完成`
- `Gemini | 发言 | 生成中`
- `Grok | 静默 | 本轮不参与`

### 原型要求

- 轨道尽量等宽
- 状态栏固定在每个轨道顶部
- 轨道正文不被控制台逻辑破坏

---

## 三、底部讨论控制台总骨架

## 1. 控制台纵向六行结构

我建议把底部控制台画成六行。

### 第 1 行：阶段总览行

显示：

- 当前阶段
- 第几轮
- 当前模式
- 本轮目标

### 第 2 行：规则与参与者行

显示：

- 常驻规则 Tag
- 临时补充 Tag
- 参与 AI 状态单元

### 第 3 行：Draft 标题行

显示：

- Draft 名称
- Draft 来源入口
- 版本状态

### 第 4 行：Draft 主编辑区

这是最大的一块。

### 第 5 行：动作行

- 左：辅助动作
- 右：唯一主按钮

### 第 6 行：反馈行

显示：

- 当前发送提示
- 当前异常
- 当前系统建议

---

## 2. 控制台标准低保真

```text
+----------------------------------------------------------------------------------+
| [首轮独立分析] [第1/4轮] [4轮标准] 目标：让各AI独立给出压缩方案                   |
| 规则: [高压缩] [只说重点] [理性独立]   临时: [只看成本]   参与: C/G/G              |
| 首轮 Draft | 来源: 核心问题/规则/模板 | 状态: 系统已生成                           |
| -------------------------------------------------------------------------------- |
| 你将看到一份完整可发送 Draft                                                     |
|                                                                                  |
| 1. 请独立分析问题                                                                |
| 2. 请先给出结论                                                                  |
| 3. 请高压缩表达，不要写长铺垫                                                    |
| 4. 你的回答会给其他 AI 阅读，请清晰排版                                          |
|                                                                                  |
| 问题：......                                                                     |
|                                                                                  |
| 用户补充：......                                                                 |
| -------------------------------------------------------------------------------- |
| [快捷文本] [常驻规则] [重生成草稿]                           [发送首轮]            |
| 将发送给 3 个 AI。当前无异常。                                                   |
+----------------------------------------------------------------------------------+
```

## 四、不同状态下的低保真原型

## 状态 A：`idle`

### 目标

- 让用户准备讨论
- 不直接面对一份已经生成的 Draft

### 视觉重心

- 讨论主题输入
- 模式选择
- 默认规则
- 参与 AI 预览
- 主按钮 `开始首轮`

### 低保真示意

```text
+----------------------------------------------------------------------------------+
| [准备开始讨论] [尚未开始] [默认模式: 4轮标准]                                     |
| 规则: [高压缩] [只说重点] [理性独立] [不要奉承]                                  |
| 参与: [ChatGPT 发言] [Gemini 发言] [Grok 发言]                                   |
| -------------------------------------------------------------------------------- |
| 讨论主题                                                                         |
| [请在这里输入这次要讨论的问题..................................................] |
|                                                                                  |
| 模式选择:  ( ) 3轮快速   (*) 4轮标准   ( ) 5轮深度                               |
|                                                                                  |
| 可选快捷开场: [首轮压缩模板] [理性讨论模板] [技术分析模板]                        |
| -------------------------------------------------------------------------------- |
| [快捷文本] [常驻规则] [调整参与者]                           [开始首轮]            |
| 系统将先生成首轮 Draft，不会直接发送。                                           |
+----------------------------------------------------------------------------------+
```

### 交互原则

- Draft 编辑区此时不是完整正文，而是准备态
- 用户点 `开始首轮` 后才进入 Draft 生成

---

## 状态 B：`draft_preparing`

### 目标

- 明确告诉用户系统正在拼本轮 Draft

### 视觉重心

- 阶段条
- Draft 状态
- 反馈行

### 低保真示意

```text
+----------------------------------------------------------------------------------+
| [首轮独立分析] [第1/4轮] [4轮标准] 目标：生成首轮工作稿                           |
| 规则: [高压缩] [只说重点] [理性独立]   参与: [ChatGPT] [Gemini] [Grok]            |
| 首轮 Draft | 来源整理中... | 状态: 生成中                                         |
| -------------------------------------------------------------------------------- |
| 正在生成本轮 Draft...                                                            |
|                                                                                  |
| - 正在装配核心问题                                                               |
| - 正在加入阶段模板                                                               |
| - 正在合并规则                                                                   |
| -------------------------------------------------------------------------------- |
| [取消]                                                       [生成中]            |
| 请稍候，系统正在准备可编辑 Draft。                                               |
+----------------------------------------------------------------------------------+
```

### 交互原则

- 主按钮不可点
- 用户能看见正在处理什么

---

## 状态 C：`draft_ready`

### 目标

- 让用户专注阅读与修改
- 准备真正发送

### 视觉重心

- Draft 正文
- Draft 来源
- 主按钮

### 低保真示意

```text
+----------------------------------------------------------------------------------+
| [交叉讨论] [第2/4轮] [4轮标准] 目标：先总结对方重点，再修正自己的判断             |
| 规则: [高压缩] [只说重点] [理性独立]   临时: [不要重复背景]                       |
| 交叉讨论 Draft | 来源: 核心问题/规则/他方摘要/用户补充 | 状态: 用户可编辑         |
| -------------------------------------------------------------------------------- |
| 下面是当前轮 Draft                                                               |
|                                                                                  |
| 请阅读其他 AI 的观点后继续独立推理。                                             |
| 先简要总结你理解到的对方核心点。                                                 |
| 再说明你同意与不同意的部分。                                                     |
| 最后给出你修正后的方案。                                                         |
|                                                                                  |
| 对方摘要：......                                                                 |
|                                                                                  |
| 用户补充：不要写长铺垫，只说真正影响决策的部分。                                 |
| -------------------------------------------------------------------------------- |
| [快捷文本] [常驻规则] [重生成草稿]                         [发送交叉讨论]          |
| 本轮将发送给 2 个发言 AI，1 个 AI 静默。                                         |
+----------------------------------------------------------------------------------+
```

### 交互原则

- 这是整个控制台最重的状态
- Draft 区占最大面积

---

## 状态 D：`round_waiting`

### 目标

- 让用户盯住“谁完成了、谁还没完成”
- 防止误以为还能继续改当前轮

### 视觉重心

- 参与者状态
- 反馈行
- 局部异常处理

### 低保真示意

```text
+----------------------------------------------------------------------------------+
| [交叉讨论] [第2/4轮] [4轮标准] 目标：等待本轮完成                                 |
| 规则: [高压缩] [只说重点]    参与: [ChatGPT 已完成] [Gemini 生成中] [Grok 静默]   |
| 交叉讨论 Draft | 来源: 已锁定 | 状态: 本轮已发送                                  |
| -------------------------------------------------------------------------------- |
| 当前 Draft 已发送，本轮编辑已锁定。                                               |
|                                                                                  |
| 发送对象: ChatGPT / Gemini                                                       |
| 完成进度: 1 / 2                                                                  |
|                                                                                  |
| 当前等待中...                                                                    |
| -------------------------------------------------------------------------------- |
| [局部重试] [跳过该AI] [提前总结]                            [等待本轮完成]         |
| Gemini 仍在生成中。Grok 本轮静默，不计入等待。                                   |
+----------------------------------------------------------------------------------+
```

### 交互原则

- Draft 主体只读
- 焦点切换到轨道状态与异常处理

---

## 状态 E：`round_review`

### 目标

- 告诉用户本轮结束了
- 让用户决定下一步

### 视觉重心

- 摘要
- 分歧/共识
- 下一步主按钮

### 低保真示意

```text
+----------------------------------------------------------------------------------+
| [本轮完成] [第2/4轮] [4轮标准] 目标：决定是否进入压缩轮                           |
| 规则: [高压缩] [只说重点]    参与: [ChatGPT 已完成] [Gemini 已完成] [Grok 静默]   |
| 本轮摘要 | 来源: 双方回复整理 | 状态: 待进入下一阶段                              |
| -------------------------------------------------------------------------------- |
| 当前共识                                                                         |
| - ......                                                                         |
| - ......                                                                         |
|                                                                                  |
| 当前剩余分歧                                                                     |
| - ......                                                                         |
| - ......                                                                         |
|                                                                                  |
| 系统建议：进入压缩轮，只讨论仍影响最终结论的部分。                               |
| -------------------------------------------------------------------------------- |
| [局部重试] [屏蔽本段输出] [提前总结]                        [进入压缩轮]           |
| 当前已具备继续推进条件。                                                         |
+----------------------------------------------------------------------------------+
```

### 交互原则

- 这里的 Draft 区可以变成“摘要视图”
- 用户关注点从编辑文本转向选择路径

---

## 状态 F：`summarizer_selecting`

### 目标

- 在进入最终轮前，明确谁负责收束

### 视觉重心

- 参与者状态
- 推荐总结者
- 主按钮

### 低保真示意

```text
+----------------------------------------------------------------------------------+
| [进入总结] [第4/4轮前] [4轮标准] 目标：确认最终总结者                             |
| 规则: [高压缩] [只说重点]                                                         |
| 总结者确认 | 来源: 投票/系统建议 | 状态: 待确认                                    |
| -------------------------------------------------------------------------------- |
| 推荐总结者：ChatGPT                                                              |
|                                                                                  |
| 参与者状态                                                                       |
| - ChatGPT：推荐总结者                                                            |
| - Gemini：最终轮静默                                                             |
| - Grok：最终轮静默                                                               |
|                                                                                  |
| 你也可以改为手动指定或双总结。                                                   |
| -------------------------------------------------------------------------------- |
| [改为手动指定] [重新投票] [双总结]                           [确认总结者]          |
| 确认后，系统将生成最终总结 Draft。                                               |
+----------------------------------------------------------------------------------+
```

### 2 AI 特殊情况

2 AI 时可以改成更直接的低保真：

```text
+---------------------------------------------------------------+
| 请选择最终总结方式                                             |
| [ChatGPT 总结]  [Gemini 总结]  [双总结]                        |
+---------------------------------------------------------------+
```

---

## 状态 G：`finished`

### 目标

- 给出最终结果
- 允许用户继续下一场

### 视觉重心

- 最终结论
- 后续动作

### 低保真示意

```text
+----------------------------------------------------------------------------------+
| [讨论完成] [第4/4轮] [4轮标准]                                                    |
| 最终结论 | 状态: 已完成                                                           |
| -------------------------------------------------------------------------------- |
| 最终决议                                                                         |
| - ......                                                                         |
|                                                                                  |
| 论证路径                                                                         |
| - ......                                                                         |
|                                                                                  |
| 遗留分歧                                                                         |
| - ......                                                                         |
|                                                                                  |
| 风险边界                                                                         |
| - ......                                                                         |
| -------------------------------------------------------------------------------- |
| [复制最终结论] [导出讨论记录] [复用规则再来一轮]             [开始新讨论]         |
| 本次讨论已结束。                                                                 |
+----------------------------------------------------------------------------------+
```

## 五、控制台区域高亮规则

不同状态下，不只是内容不同，视觉强调也应不同。

## 1. `idle`

### 需要高亮

- 主题输入区
- 模式选择
- `开始首轮`

### 需要弱化

- Draft 区

---

## 2. `draft_ready`

### 需要高亮

- Draft 正文区
- Draft 来源入口
- 主按钮

### 需要弱化

- 反馈行

---

## 3. `round_waiting`

### 需要高亮

- 参与者状态
- 异常提示
- 等待进度

### 需要弱化

- Draft 文本本体

---

## 4. `round_review`

### 需要高亮

- 本轮摘要
- 分歧 / 共识块
- 下一阶段主按钮

---

## 5. `summarizer_selecting`

### 需要高亮

- 推荐总结者
- 参与者角色变化
- `确认总结者`

---

## 六、控制台只读与可编辑规则

## 可编辑状态

- `idle`
- `draft_ready`

## 半可编辑状态

- `summarizer_selecting`

这里允许切换总结者，但不一定允许再改上一轮正文。

## 只读状态

- `draft_preparing`
- `round_dispatching`
- `round_waiting`
- `finished`

### 原型要求

只读时要明显表现出来：

- 背景弱化
- 输入光标消失
- 显示“已锁定”或“本轮已发送”

---

## 七、V1 原型中不需要画得很复杂的部分

为了保持原型清晰，以下内容在 V1 低保真里可以先弱化：

- 每个 AI 的独立 Draft 差异视图
- 完整历史引用来源树
- 复杂权限切换
- 细粒度系统日志
- 动画细节

原型只要先把主骨架与状态切换讲清楚就够了。

---

## 八、最推荐的出图顺序

如果接下来要画原型，我建议按这个顺序画。

### 1. 先画整页总骨架

- 顶部总览
- 中部轨道
- 底部控制台

### 2. 再画 5 个关键状态

- `idle`
- `draft_ready`
- `round_waiting`
- `round_review`
- `summarizer_selecting`

### 3. 最后补完成态

- `finished`

这样最省力，也最容易让团队先达成一致。

---

## 九、结论

我对这份低保真原型说明的核心判断是：

- 它应该帮助我们把“讨论控制台”从抽象概念变成可画的界面
- 它必须突出 Draft 区和唯一主按钮
- 它必须让不同状态下的视觉重心变化明确
- 它不需要很花，但必须极清楚

如果继续往下，我建议下一步直接进入：

- `规则 Tag 与快捷文本管理规范`

或者更直接一点：

- 开始做第一版真实界面实现
