# SchemeChat 讨论工作台低保真线框说明 V2

## 文档目的
本文档把新的工作台信息架构收敛成可直接落原型的线框说明。

这版线框统一遵循：

- Notion 式暖白、轻边界
- Cal.com 式清晰骨架
- Linear 式紧凑状态表达

本文档不讨论视觉细节数值，只描述版式和区域关系。

---

## 一、线框原则

### 1. 工作台是浮动工作区，不是底部整条控制条
主界面仍然以 AI 官方网页为主。

工作台负责“本轮准备与推进”，因此必须：

- 让 Draft 成为中心
- 让状态带够薄
- 让底部动作始终看得见

### 2. 先保核心路径，再容纳次级信息
工作台的第一职责不是“信息完整”，而是“当前轮能顺利发出去”。

因此线框优先保证：

- 轮次状态可见
- Draft 可编辑
- 主按钮可点击

### 3. 不再坚持绝对无局部滚动
在高度紧张时，允许右侧辅助区成为唯一内部滚动区。

## 二、收起态 Launcher 线框

```text
+--------------------------------------------------------------------------------+
| [准备开始] [4 轮标准] [待生成]                                 [展开工作台]     |
|                                                                                |
| 讨论主题 / Draft 快速编辑                                                       |
| [..........................................................................]   |
|                                                                                |
| 可生成首轮 Draft                                       [临时对话] [同步] [开始首轮] |
+--------------------------------------------------------------------------------+
```

### 说明
- 收起态只承担快速推进
- 不放来源、参与者、长规则
- 主按钮与同步、临时对话保持同一组

## 六、交互高亮规则

### 1. `idle`
- 高亮：主题输入、模式切换、主按钮
- 弱化：来源、更多操作

### 2. `draft_ready`
- 高亮：Draft 编辑区、重新生成、主按钮
- 弱化：模式说明长文

### 3. `round_waiting`
- 高亮：参与 AI 状态、反馈条
- Draft 改为只读视觉

### 4. `round_review`
- 高亮：总结摘要、下一步主按钮
- 右列展示本轮共识 / 分歧摘要

---

## 七、线框中的文案预算规则

- launcher 里不再出现长句状态，优先写成 `可生成首轮 Draft`
- footer 状态优先写结果，不写教学说明
- `重新生成` 优先于 `重生成`
- 参与者 chip 只保留“名称 + 当前状态”，不叠加额外发送态
- disclosure 入口优先写成 `Draft 来源`、`更多操作` 这类短标签

详细规则见：

- `product/discussion_console_copy_guidelines_2026-04-08.md`

---

## 八、这版线框解决什么问题

这版线框的目标非常明确：

- 不再让右下角主按钮与 resize handle 打架
- 不再让右列卡片把 footer 顶没
- 不再让标题、状态、说明重复堆叠
- 不再让低频信息抢 Draft 的空间

工作台应该像一个“轻量、安静、可推进”的本轮工作区，而不是一个塞满模块的说明面板。
