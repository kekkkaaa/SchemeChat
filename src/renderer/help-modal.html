场景速记

        
只用 SchemeChat.exe 本体：什么都不用配，直接双击 exe。

        
Exe + Codex：启动 SchemeChat 后，配置一次 ~/.codex/config.toml。

        
Exe + Claude Code（无项目）：启动 SchemeChat 后，执行一次 claude mcp add ...。

        
项目 + Claude Code：可以用项目根目录里的 .mcp.json，也可以继续用全局命令。

      


      

        
推荐用法

        
默认主路：让 Codex / Claude Code 优先走讨论流程工具，不要把 inject -> submit -> capture 当成完整多轮编排主路。

        
推荐顺序是：先用 get_discussion_flow_state 读状态，再用 update_discussion_flow 改 topic / 本轮补充 / draft / runMode，最后用 trigger_discussion_action 触发开始、发送、继续、下一步等动作。

        
写操作工具默认关闭。如需 update_discussion_flow、trigger_discussion_action 等能力，请先在应用 Settings 中开启 MCP Write Tools。这个开关会持久保存，之后正常双击 exe 启动也会按该设置生效。

        
get_discussion_flow_state 用来读当前轮次、Draft、推进模式和下一步可执行动作。

        
update_discussion_flow 用来改 topic / 本轮补充 / draft / runMode，遵守和界面一致的可编辑规则。

        
trigger_discussion_action 会复用 SchemeChat 现有的自动/手动讨论流程，由应用自己负责发送、等待、暂停、补抓和下一轮推进。

        
长时间自动流程建议把 waitForCompletion 设为 false，然后继续用状态工具观察流程进度。

        
低层工具定位：inject_text_to_panes、submit_message_to_panes、capture_latest_replies 更适合调试、补充上下文或人工干预，不建议直接拿来自己拼完整 round loop。

      


      
      

        
可用 MCP 工具

        

          

            get_workspace_snapshot
            查看所有 AI 面板布局与状态
          

          

            inspect_round_status
            低层调试：检查各面板当前是否正在回复
          

          

            capture_latest_replies
            低层调试：抓取各面板最新 AI 回复内容
          

          

            get_discussion_flow_state
            主路：查看讨论流程状态、当前 Draft 和下一步动作
          

          

            update_discussion_flow
            主路：更新 topic、本轮补充、draft 或 runMode
          

          

            trigger_discussion_action
            主路：触发开始、发送、继续、下一步等高层讨论动作
          

          

            inject_text_to_panes
            低层调试：向指定面板输入框直接写入文字
          

          

            submit_message_to_panes
            低层调试：直接触发发送
          

          

            broadcast_context_to_panes
            低层调试：广播补充上下文，可选自动发送
